(Context level) Readme for 30150-.ftr.module.ts

Type Angular Routing module
Recomended folder /src/app/features/{this-is-my-feature}
Generator group Feature level

Before continuing

Remark

Generators are divided into three groups.
  • The first group has a Context level.
  • The second one is a View-Model level.
  • The third one is a Feature level.

Context level code is generated once and shared by other parts of the application.
The View-Model level code must be regenerated for each view you select. That is, each view you select must have its own copy of the generated code.
The View-Model level code generated for one view cannot be used by another view.
For example, the product attributes input form is View-Model level code
Feature level code must be regenerated for each feature you select.

Each generator has a title which starts with 5 digits.
If the first two digits are zeros it is Context level generator for Wizards
If one the first two digits is non zero it is View-Model level generator for Wizards If one the first two digits is non zero it is Feature level generator for Wizards

The currently selected generator is a Feature level code generator.

CLI-related information
Suppose your view has a name ThisIsMyView
Inside your Angular project
1. Run the command

    
            ng g module ThisIsMyFeatureFtr --routing
2. It will create the folder this-is-my-feature-ftr and inside create the set of files this-is-my-feature-ftr.module.ts and this-is-my-feature-ftr.routing.module.ts
3. In Visual Studio right-click created fordel and generate the component-body It updates this-is-my-feature-ftr.module.ts only.

Project-related information
Step 0: Do not run the command described under CLI-related information. It's for information purpose only and makes you hint about the type of component you are going to generate
Step 1: Inside folder src/app create the folder 'features' (You can choose any folder name you like: 'myfeatures' or 'mycode' and so on)
Step 2: Inside folder of step 1 (recommended 'features') create the folder 'this-is-my-feature' (You can choose any folder name you like: 'myfeaturefolder' or 'fooooo' and so on)
Step 4: In Visual Studio right-click the 'this-is-my-feature'-folder that you created in step 2 and generate the following items

  • 30130-.ftr.module.ts
Don't forget to click save-button. It saves the code into the file and adds some information into repository of the Wizards. Repository information is actively used by all other generators that you will run later.
